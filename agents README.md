# Роли агентов

Агенты — это специализированные роли сопровождения. Оркестратор распределяет
задачи между ними, а окончательные решения сверяются с `AGENTS.md`.

## Product and Domain Agent

Владеет `docs/PROJECT.md` и критериями релевантности. Уточняет ключи, стадии,
законы и бизнес-ценность. Не меняет технические интервалы без SRE-оценки.

## Source Integration Agent

Владеет адаптерами источников и расширением браузера. Использует официальный
RSS/API, фиксирует контракт данных и добавляет образцы для тестов. Не включает
источник в работу без проверки на реальной площадке.

## Latency and SRE Agent

Владеет `docs/SLO.md`, heartbeat, метриками, запуском и восстановлением.
Разделяет внешнюю задержку площадки и внутреннюю задержку проекта.

## Notification Agent

Владеет Telegram/email и форматом сообщения. Гарантирует независимость каналов,
отсутствие дублей и сохранность закупки при ошибке доставки.

## QA and Security Agent

Проверяет тесты, секреты, негативные сценарии и соблюдение ограничений площадок.
Блокирует релиз, если секрет попал в Git, тесты не проходят или критический путь
не имеет измеримого результата.

## Delivery Guardian

Владеет outbox, состояниями каналов, retry/backoff и подтверждением Telegram.
Не допускает, чтобы успешный email маскировал неуспешный Telegram.

## Release Gatekeeper

Применяет `docs/TESTING.md`, проверяет миграции и запрещает рабочее обновление
при красном критическом сценарии.

## Оркестратор

Декомпозирует задачи, назначает владельцев с непересекающимися областями,
собирает результаты и отвечает за итоговую проверку проекта.
