# Тестирование и релиз

## Текущее покрытие

Автоматические тесты проверяют базовую фильтрацию, исключающие слова,
дедупликацию URL, backoff и формат сообщения ЕИС. Этого недостаточно для
гарантированной эксплуатации.

## Обязательные уровни

### Unit

- разбор RSS и полей ЕИС;
- релевантность без служебных параметров поиска;
- переходы состояний outbox;
- backoff и исчерпание повторов;
- независимость Telegram и email.

### API

- авторизация intake и heartbeat;
- некорректный JSON и неизвестный источник;
- первая загрузка без уведомлений;
- повторный URL;
- Telegram success, timeout, 429 и 500.

### End-to-end

Синтетический RSS должен пройти путь:

`RSS -> расширение -> intake -> matching -> SQLite -> Telegram ACK`.

Проверяются одиночная запись и всплески 10/50 записей, включая релевантную
позицию в начале, середине и конце списка.

### Отказоустойчивость

- один RSS недоступен;
- Telegram временно недоступен;
- SMTP зависает;
- backend перезапускается;
- браузер останавливается и запускается;
- компьютер засыпает и просыпается.

## Release gate

Перед рабочим обновлением обязательны:

1. Все автоматические тесты зелёные.
2. Python и JavaScript проходят синтаксическую проверку.
3. Секреты отсутствуют в Git.
4. Создана копия SQLite и конфигурации.
5. Выполнен canary intake без отправки старых записей.
6. Telegram подтвердил тестовую доставку.
7. После релиза heartbeat наблюдается минимум 30 минут.

## Запрещённый релиз

Релиз блокируется, если нет повтора неуспешной Telegram-доставки, не пройдена
миграция БД, расширение не перезагружено или не виден heartbeat.
