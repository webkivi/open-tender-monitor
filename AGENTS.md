# AGENTS.md

Этот файл задаёт обязательные правила для всех агентов, работающих с проектом.

## Цель

Обнаруживать релевантные закупки и доставлять их в Telegram не позднее
60 секунд после появления записи в RSS источника.

## Источники истины

1. `docs/PROJECT.md` — продуктовая цель и критерии готовности.
2. `docs/SLO.md` — определения задержек и целевые показатели.
3. `docs/ARCHITECTURE.md` — границы компонентов и поток данных.
4. `config/sources.json` — ключи и настройки источников.
5. Автоматические тесты — подтверждение поведения кода.

Если документы расходятся с кодом, агент обязан выявить расхождение и обновить
либо код, либо документ в рамках той же задачи.

## Обязательные правила

- Не обходить CAPTCHA, анти-DDoS и ограничения площадок.
- Использовать официальный RSS/API, если он доступен.
- Не применять ротацию IP для обхода блокировок.
- Не выводить токены, пароли и SMTP-реквизиты в логи, ответы или документацию.
- Не отправлять старые записи при первом подключении нового RSS.
- Не считать служебные параметры RSS частью текста закупки.
- Не уведомлять повторно по уже известному URL.
- Не заявлять поддержку площадки, пока нет рабочего адаптера и проверки.
- Изменения в критическом пути `RSS -> Telegram` сопровождать тестом.
- После изменения расширения явно указывать, что его нужно перезагрузить.

## Критический путь

`ЕИС RSS -> расширение Яндекс.Браузера -> /api/intake/eis -> SQLite ->
Telegram Bot API`

Все изменения сначала оцениваются по влиянию на этот путь и SLO.

## Рабочий процесс агента

1. Прочитать релевантные документы.
2. Проверить текущее состояние и не предполагать, что сервис запущен.
3. Сформулировать измеримый ожидаемый результат.
4. Внести минимальное связное изменение.
5. Запустить тесты и проверку здоровья.
6. Обновить документацию, если изменились контракт или эксплуатация.
7. Сообщить фактический результат и остаточный риск.

## Владение

Роли и их зоны ответственности определены в `agents/README.md`. Один агент
может исполнять несколько ролей, но должен явно указать, в какой роли принимает
решение.
