# Open Tender Monitor

Локальный сервис раннего обнаружения закупок. Текущий рабочий контур получает
официальные RSS-потоки ЕИС через Яндекс.Браузер, фильтрует закупки по предмету
и ключевым словам, удаляет дубли и отправляет новые совпадения в Telegram.

Главная цель продукта: доставить уведомление не позднее 60 секунд после
появления закупки в RSS ЕИС. Задержка между публикацией закупки на площадке и
её появлением в RSS измеряется отдельно, поскольку управляется ЕИС.

## Текущее состояние

- ЕИС 44-ФЗ и 223-ФЗ: работает через официальные RSS-подписки.
- Фильтр: название и поле «Наименование объекта закупки».
- Стадия: RSS должен быть создан только для «Подача заявок».
- Telegram: основной рабочий канал.
- Email: резервный канал, требует повторной проверки SMTP.
- RTS-Tender, ЕАТ «Берёзка», Росэлторг: ещё не введены в эксплуатацию.
- Интервал браузерного опроса: 45 секунд после обновления расширения.

## Запуск

```bash
cd /mnt/c/Users/user/Documents/Парсер/open-tender-monitor
./scripts/start.sh
```

Остановка:

```bash
./scripts/stop.sh
```

Панель внутри WSL доступна на порту `8081`. Актуальный адрес можно увидеть в
логе запуска. Расширение находится в `yandex-eis-rss-bridge`.

## Проверка

```bash
. .venv/bin/activate
python -m unittest discover -s tests -v
python -m py_compile app.py tender_monitor/*.py
node --input-type=module --check < yandex-eis-rss-bridge/background.js
node --check yandex-eis-rss-bridge/options.js
```

Секреты находятся только в `.env` и
`yandex-eis-rss-bridge/private-config.js`; оба файла исключены из Git.
