# Архитектура

## Поток данных

```text
ЕИС
  -> официальный RSS
  -> расширение Яндекс.Браузера
  -> POST /api/intake/eis
  -> проверка стадии и ключей
  -> SQLite / дедупликация URL
  -> очередь уведомлений
  -> Telegram и email (независимо)
```

## Компоненты

### Расширение Яндекс.Браузера

Каталог: `yandex-eis-rss-bridge`.

Отвечает за сетевой доступ к ЕИС через профиль браузера без VPN. Раз в
45 секунд параллельно читает все RSS, объединяет записи и передаёт их в
локальный сервис. При первом чтении нового RSS отправляет записи в тихом
режиме для формирования исходной базы.

### Веб-приложение

Файл: `app.py`.

Принимает записи и контрольные сигналы расширения, проверяет токен связи,
применяет правила источника и сохраняет закупку. Отправка не выполняется в HTTP-запросе:
для каждого канала создаётся отдельное задание очереди.

### Доменная логика

- `tender_monitor/sources.py` — ключи и релевантность.
- `tender_monitor/presentation.py` — разбор ЕИС и формат сообщения.
- `tender_monitor/notifiers.py` — Telegram и email.
- `tender_monitor/storage.py` — SQLite, дубли, состояния и метрики.
- `tender_monitor/service.py` — планировщик обычных источников.

### Хранилище

SQLite хранит закупки, события, состояние источников, историю проверок и очередь
`notification_jobs`. У задания есть канал, число попыток, время следующей попытки,
ошибка и подтверждённое время доставки.
Уникальность URL обеспечивает дедупликацию.

Telegram является срочным каналом: только его подтверждённая отправка заполняет
`tenders.notified_at`. Email обрабатывается отдельным заданием и не задерживает Telegram.
При временной ошибке доставка Telegram повторяется через 5, 15, 45, 120 и 300 секунд;
после шести неудач задание фиксируется как `failed` для разбора.

## Границы задержки

Система не управляет интервалом между публикацией закупки и появлением записи
в RSS ЕИС. Управляемая часть начинается с появления записи в RSS и заканчивается
подтверждённой отправкой Telegram.

## Отказы

- Недоступен один RSS: остальные продолжают обрабатываться.
- Недоступен ЕИС: расширение фиксирует ошибку, обход защиты не выполняется.
- Недоступен Telegram: закупка остаётся в базе с ошибкой доставки.
- Не работает SMTP: Telegram должен продолжать работать независимо.
- Перезапуск WSL: адрес локального сервиса может измениться; расширение требует
  проверки адреса.
